# Content Guide

## Global voice

- Minimal, direct, no fluff.
- Short sentences.
- Prefer concrete nouns and outcomes.
- Avoid em dashes.

## Home page content model

Home page is a grid of project tiles that matches the wireframe layout:
- One featured wide tile at the top.
- Two tiles in the middle with different aspect ratios.
- One wide tile near the bottom.
Each tile includes:
- Title (required)
- Subtext (short descriptor, required)

Subtext format suggestion
- “Year, Role” or “Product area, year”
Keep it short.

## About page content model

Two-column layout:
Left column
- Email
- LinkedIn
- GitHub
- Resume PDF link

Right column
- Short bio paragraph(s)

Keep the bio short. Aim for 3–6 lines per paragraph.

## Case study page content model

Required fields
- Hero image
- Project title
- Tagline (short description)
- Platform
- Role(s)
- Year
- Tools
- Overview paragraph
- Up to 3 metrics
- Body content (images + captions + short text blocks)

Suggested section order (matches wireframe rhythm)
1. Hero
2. Title + overview + metadata (platform, roles, year, tools)
3. Divider
4. Metrics row (up to 3)
5. Divider
6. Body content blocks
   - Alternating image sizes is encouraged
   - Captions are required when the image needs context
7. Closing paragraph or short wrap-up
8. Footer

Metrics rules
- 1–3 metrics
- Format: “##” headline with a short label under it
- Keep labels short (2–5 words)

Image captions
- Describe what the reader should notice
- Do not repeat the title
- Keep it one sentence

Alt text
- Describe what is visible and why it matters.
- If the image is purely decorative, use empty alt.
